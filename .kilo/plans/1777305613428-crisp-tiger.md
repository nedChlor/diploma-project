# План реализации функционала переводчика с Gemini API

## Обзор
Проект "LexAI" имеет готовый UI для переводчика (index.html, app.js с темой, style.css), но отсутствует логика перевода. Необходимо реализовать бэкенд с использованием Google Gemini API для перевода текста между русским, английским и казахским языками, включая базовый анализ лексики. План включает установку зависимостей, настройку API, эндпоинты и фронтенд-интеграцию.

## Предварительные требования
- Установка Python 3.8+ для бэкенда.
- Получение API-ключа Google Gemini (через Google AI Studio или Cloud Console).
- Установка Node.js (если потребуется для фронтенда, но сейчас не нужно).
- Создание виртуального окружения Python (`venv`).

## Шаги реализации

### 1. Настройка бэкенда (backend/main.py)
- **Установка зависимостей**:
  - Создать `backend/requirements.txt` со следующим содержимым:
    ```
    flask==3.0.0
    flask-cors==4.0.0
    google-generativeai==0.5.0
    python-dotenv==1.0.0З
    ```
  - Выполнить `pip install -r requirements.txt` в виртуальном окружении.
- **Конфигурация**:
  - Создать `.env` файл в `backend/` с переменной `GEMINI_API_KEY=ваш_ключ`.
  - Настроить Flask-приложение с CORS для запросов с фронтенда.
- **Эндпоинт перевода**:
  - Создать POST `/translate`: принимает JSON с `text`, `from_lang`, `to_lang`.
  - Интегрировать Gemini API: использовать модель `gemini-1.5-flash` для генерации перевода и анализа лексики (например, "Translate from {from_lang} to {to_lang} and provide lexical analysis: {text}").
  - Обработка ошибок: невалидный текст, API-ошибки, ограничения (rate limits).
  - Возврат JSON: `{ "translation": "...", "analysis": "..." }`.
- **Безопасность**: API-ключ не должен храниться в коде; использовать `.env` и `os.getenv()`.

### 2. Настройка фронтенда (frontend/app.js)
- **Добавление логики перевода**:
  - Добавить event listener на `#translateBtn`: собирать данные из `#inputText`, `#langFrom`, `#langTo`.
  - Валидация: проверить, что текст введён (не пустой).
  - Отправка запроса: использовать `fetch()` POST на бэкенд (например, `http://localhost:5000/translate`).
  - Обработка ответа: заполнить `#outputText` переводом, `#analysisResult` анализом лексики (HTML для списка слов/определений).
- **UI/UX улучшения**:
  - Добавить индикатор загрузки (spinner на кнопке, disable состояния).
  - Обработка ошибок: показывать сообщения в UI (например, в новом div под кнопкой).
  - Доступность: обновлять `aria-live` для результатов.
- **Интеграция**: Запустить бэкенд на порту 5000, фронтенд на localhost (через сервер или просто открыть index.html).

### 3. Дополнительные файлы и конфигурации
- **requirements.txt**: Список Python-зависимостей.
- **.env**: Для API-ключа (добавить в .gitignore).
- **Обновление index.html**: Добавить loading-spinner (CSS в style.css) или error-message div.
- **Документация**: Обновить README.md с инструкциями по запуску (если есть).

### 4. Тестирование и валидация
- **Бэкенд**: Тестировать эндпоинт с Postman или curl; проверить перевод и анализ на разных языках.
- **Фронтенд**: Тестировать полный поток: ввод текста → клик → результат в output.
- **Интеграция**: Запустить оба сервера, проверить CORS и ошибки.
- **Ограничения**: Проверить rate limits Gemini (15 запросов/мин бесплатно), обработка больших текстов.
- **Кросс-браузер**: Тестировать в Chrome/Firefox.

## Риски и соображения
- **API-лимиты**: Бесплатный тариф Gemini имеет ограничения; рассмотреть paid-план при нагрузке.
- **Языковая поддержка**: Gemini хорошо работает с ru/en/kz, но проверить точность перевода.
- **Анализ лексики**: Начать с простого (определения слов), расширить позже (частота, синонимы).
- **Безопасность**: Не логировать API-ключ; использовать HTTPS в продакшене.
- **Производительность**: Для больших текстов оптимизировать промпты Gemini.
