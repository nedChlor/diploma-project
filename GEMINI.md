# Техническое задание: LexAI — веб-приложение для перевода и анализа лексики

## Описание проекта
Дипломный проект студента колледжа по специальности "Программное обеспечение".
Веб-приложение для интеллектуального перевода текста и анализа лексики с использованием AI.

## Стек технологий
- Frontend: HTML, CSS, JavaScript (без фреймворков)
- Backend: Python, FastAPI
- База данных: Firebase Firestore
- Аутентификация: Firebase Authentication
- AI: Google Gemini API (gemini-2.0-flash)
- Хостинг frontend: Vercel
- Хостинг backend: Render

## Дизайн
- Стиль: светлый, чистый, современный
- Цветовая схема: белый фон (#f5f7ff), акцент фиолетовый (градиент #6366f1 → #8b5cf6)
- Шрифт: Segoe UI
- Карточки с тенями, скруглённые углы (border-radius: 24px)
- Sticky header с логотипом LexAI и навигацией

## Функционал

### Страница переводчика (index.html)
- Два поля textarea: левое (ввод), правое (результат, readonly)
- Два select для выбора языка: русский, английский, казахский
- Кнопка перевода → отправляет запрос на backend
- Кнопка "Анализ лексики" → показывает блок анализа под переводчиком
- Блок анализа отображает: уровень CEFR, стиль текста, сложные слова, ключевые слова
- Кнопка "Сохранить" → сохраняет перевод в Firestore (только для авторизованных)

### История (history.html)
- Список сохранённых переводов из Firestore текущего пользователя
- Каждая карточка: исходный текст, перевод, языки, дата
- Кнопка удаления записи

### О проекте (about.html)
- Описание проекта, технологий, автора
- Автор: Раимбердиев Давлят Фархадович

### Аутентификация
- Firebase Auth: регистрация и вход по email/паролю
- Модальное окно входа/регистрации
- В header: кнопка "Войти" если не авторизован, имя пользователя + "Выйти" если авторизован

## Backend (FastAPI)

### Эндпоинты:
POST /translate
- Принимает: { text, from_lang, to_lang }
- Возвращает: { translation }

POST /analyze  
- Принимает: { text, lang }
- Возвращает: { level, style, hard_words[], key_words[] }

### Системный промпт для Gemini:
Ты — языковой ассистент LexAI. Выполняешь только перевод и анализ текста.
Поддерживаемые языки: русский, английский, казахский.
При переводе возвращай только перевод без пояснений.
При анализе возвращай строго JSON: { level, style, hard_words[], key_words[] }

## Важные требования
- API ключи хранятся только в .env на backend, frontend их не видит
- .env и __pycache__ добавлены в .gitignore
- Backend принимает запросы только с разрешённых origins (CORS)
- Все запросы к Gemini идут только через backend
- Код писать чисто, с комментариями на русском языке
- Не использовать сторонние CSS фреймворки (только чистый CSS)
